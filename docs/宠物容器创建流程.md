# 宠物容器创建流程

## 部署架构说明

- **anyclaw-manager**：API + 前端合一的镜像，单容器运行，端口 8080。前端静态资源由 API 内置提供。
- **宠物容器**：在「服务器」中配置的 Host 上，通过 SSH 执行 `docker run` 启动的实例（默认镜像 anyclaw/anyclaw）。每个领养的宠物对应一个独立容器。

领养时，运行中的 anyclaw-manager 会 SSH 到配置的 Host，在其上启动宠物容器。

---

## 一、调用链

```
用户点击「领养」 → POST /instances { name }
  → instances.Create
    → 1. DeductUserEnergy(100) 扣除用户电量
    → 2. CreateInstance 写入 DB，status='creating'
    → 3. go func() { scheduler.Run(...) } 异步启动容器
    → 4. 立即返回 201 + instance
```

## 二、Scheduler.Run 内部

```
scheduler.Run(ctx, instanceID, token)
  → ListEnabledHosts() 从 hosts 表取 enabled=1 的服务器
  → 若为空：返回 "no enabled hosts configured"
  → 取第一个 host，用 host.DockerImage 或默认 anyclaw/anyclaw
  → runSSH(host, "docker run -d -e ANYCLAW_API_URL=... -e ANYCLAW_INSTANCE_ID=... -e ANYCLAW_TOKEN=... image")
  → 成功：UpdateInstanceContainer(id, containerID, hostID)，status='running'
  → 失败：UpdateInstanceStatus(id, "error")
```

## 三、常见失败原因

| 失败点 | 原因 | 处理 |
|--------|------|------|
| no enabled hosts | hosts 表为空或全部 enabled=0 | 管理员在「服务器」页面添加并启用 |
| ssh_key or ssh_password required | 添加时未填 SSH 密码或私钥 | 编辑服务器，填写密码或私钥 |
| ssh dial | 网络不通、端口错误、认证失败 | 检查 addr、ssh_port、密码/私钥 |
| docker run 失败 | 目标机无 Docker、镜像拉取失败 | 在目标机执行 `docker run -d anyclaw/anyclaw` 测试 |
| API URL 错误 | 容器内无法访问 API | 配置 ANYCLAW_API_URL 为容器可访问的地址（非 localhost） |

## 四、配置检查

1. **ANYCLAW_API_URL**：容器启动时需能访问此地址。若 API 在 103.74.192.209:8080，应设为 `http://103.74.192.209:8080`（或公网域名）。若未配置，默认 `http://localhost:8080`，容器内无法访问。
2. **hosts 表**：至少一条 enabled=1，且 ssh_key 或 ssh_password 必填其一。
3. **Docker 镜像**：host 可指定 docker_image，否则用默认 anyclaw/anyclaw。

## 五、日志排查

API 启动后，领养时会在日志中看到：

- `[instances] created instance id=X name=...`：实例已创建
- `[scheduler] no enabled hosts in DB`：无可用服务器
- `[scheduler] instance X: using host ...`：选中了调度主机
- `[scheduler] ssh docker run on ... failed`：SSH 或 docker run 失败
- `[instances] scheduler.Run failed for instance X`：调度失败
- `[instances] instance X container started`：容器创建成功
