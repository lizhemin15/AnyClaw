# AnyClaw 生产部署流程

## 一、整体流程概览

```
开发 → 推送代码 → GitHub Actions 构建 → 拉取镜像 → 部署运行
```

本地无需安装 Go、Node、Docker，所有构建在 GitHub Actions 完成。

---


## 二、推送与构建

### 1. 推送代码触发构建

```bash
git add .
git commit -m "your message"
git push origin main
```

推送后自动触发：
- **Build All**：构建 Server、API、Web 二进制/静态文件（Artifacts）
- **Docker Build**：构建并推送 Docker 镜像到 GHCR

### 2. 打 Tag 发布版本（可选）

```bash
git tag v1.0.0
git push origin v1.0.0
```

同样会触发 Docker 构建，镜像会打上 `v1.0.0` 标签。

### 3. 手动触发

GitHub 仓库 → **Actions** → 选择工作流 → **Run workflow**

### 4. 镜像地址

构建完成后，镜像推送到 GitHub Container Registry：

```
ghcr.io/<你的用户名>/anyclaw-web:latest    # 推荐：API+Web 合一
ghcr.io/<你的用户名>/anyclaw-api:latest
ghcr.io/<你的用户名>/anyclaw-server:latest
```

将 `<你的用户名>` 替换为 GitHub 用户名或组织名。

---

## 三、部署前准备

### 1. MySQL 数据库

在任意可访问的 MySQL 8.0+ 上创建库：

```sql
CREATE DATABASE anyclaw CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 2. 服务器（运行容器的节点）

若使用 SSH 调度，需准备至少一台可 SSH 的 Linux 服务器，并安装 Docker。管理员首次登录后在「服务器」页面配置。

---

## 四、部署运行

### 方式 A：单机 Docker（最简单）

```bash
# 1. 登录 GHCR（首次需要）
docker login ghcr.io -u <你的GitHub用户名>

# 2. 拉取镜像
docker pull ghcr.io/<你的用户名>/anyclaw-web:latest

# 3. 运行（挂载卷持久化配置）
docker run -d \
  --name anyclaw \
  -p 8080:8080 \
  -v anyclaw-data:/data \
  --restart unless-stopped \
  ghcr.io/<你的用户名>/anyclaw-web:latest
```

### 方式 B：指定公网访问地址

若需从外网访问，需设置 `ANYCLAW_API_URL`，供容器内的实例连接 API：

```bash
docker run -d \
  --name anyclaw \
  -p 8080:8080 \
  -v anyclaw-data:/data \
  -e ANYCLAW_API_URL=https://your-domain.com \
  --restart unless-stopped \
  ghcr.io/<你的用户名>/anyclaw-web:latest
```

### 方式 C：配合 MySQL 与反向代理

```bash
# 若已通过 setup 配置过，可用环境变量跳过向导
docker run -d \
  --name anyclaw \
  -p 8080:8080 \
  -v anyclaw-data:/data \
  -e ANYCLAW_DB_DSN="user:pass@tcp(mysql-host:3306)/anyclaw?parseTime=true&charset=utf8mb4" \
  -e ANYCLAW_API_URL=https://your-domain.com \
  --restart unless-stopped \
  ghcr.io/<你的用户名>/anyclaw-web:latest
```

## 五、首次使用

1. 访问 `http://你的服务器:8080`（或配置的域名）
2. 进入**初始化向导**，填写 MySQL 连接信息
3. 创建**管理员账号**
4. 按提示**重启容器**：`docker restart anyclaw`
5. 管理员登录后，在「服务器」中配置 SSH 主机（用于运行实例容器）

---

## 六、更新部署

```bash
# 1. 拉取最新镜像
docker pull ghcr.io/<你的用户名>/anyclaw-web:latest

# 2. 停止并删除旧容器
docker stop anyclaw
docker rm anyclaw

# 3. 用新镜像启动（保持相同参数和卷）
docker run -d \
  --name anyclaw \
  -p 8080:8080 \
  -v anyclaw-data:/data \
  --restart unless-stopped \
  ghcr.io/<你的用户名>/anyclaw-web:latest
```

或使用 `docker compose` 管理，便于一键更新。

---

## 七、常见问题

| 问题 | 处理 |
|------|------|
| 镜像拉取 404 | 检查仓库是否私有，GHCR 需登录且仓库允许 packages 读取 |
| 首次访问无响应 | 确认 8080 端口已开放，防火墙/安全组放行 |
| 配置丢失 | 必须挂载 `-v anyclaw-data:/data`，否则重启后配置丢失 |
| 实例创建失败 | 管理员需在「服务器」中添加并启用至少一台 SSH 主机 |

## 八、推荐生产架构

```
                    ┌─────────────────┐
                    │  Nginx / Caddy   │  ← 反向代理、HTTPS
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │  anyclaw-web    │  ← 8080
                    │  (API + 前端)   │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              │              │              │
       ┌──────▼──────┐ ┌─────▼─────┐ ┌─────▼─────┐
       │   MySQL     │ │  Host 1   │ │  Host 2   │  ← SSH 调度
       │             │ │ (Docker)  │ │ (Docker)  │
       └─────────────┘ └───────────┘ └───────────┘
```

- anyclaw-web 与 MySQL 可同机或分机部署
- 实例容器通过 SSH 调度到 Host 1、Host 2 等节点运行
