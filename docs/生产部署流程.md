# AnyClaw 生产部署流程

## 一、整体流程概览

```
开发 → 推送代码 → GitHub Actions 构建 → 拉取镜像 → 部署运行
```

本地无需安装 Go、Node、Docker，所有构建在 GitHub Actions 完成。

---


## 二、推送与构建

### 1. 推送代码触发构建

```bash
git add .
git commit -m "your message"
git push origin main
```

推送后自动触发：
- **Build All**：构建 Server、API、Web 二进制/静态文件（Artifacts）
- **Docker Build**：构建并推送 Docker 镜像到 Docker Hub

### 2. 打 Tag 发布版本（可选）

```bash
git tag v1.0.0
git push origin v1.0.0
```

同样会触发 Docker 构建，镜像会打上 `v1.0.0` 标签。

### 3. 手动触发

GitHub 仓库 → **Actions** → 选择工作流 → **Run workflow**

### 4. 镜像地址

构建完成后，镜像推送到 Docker Hub：

```
<你的DockerHub用户名>/anyclaw-manager:latest   # API + 前端合一，单端口部署
```

将 `<你的DockerHub用户名>` 替换为 Docker Hub 用户名。

---

## 三、部署前准备

### 1. MySQL 数据库

在任意可访问的 MySQL 8.0+ 上创建库：

```sql
CREATE DATABASE anyclaw CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 2. 服务器（运行容器的节点）

若使用 SSH 调度，需准备至少一台可 SSH 的 Linux 服务器，并安装 Docker。管理员首次登录后在「服务器」页面配置。

---

## 四、部署运行

### 方式 A：单机 Docker（最简单）

```bash
# 1. 登录 Docker Hub（首次需要，公开镜像可跳过）
docker login -u <你的DockerHub用户名>

# 2. 拉取镜像
docker pull <你的DockerHub用户名>/anyclaw-manager:latest

# 3. 运行（挂载卷持久化配置）
docker run -d \
  --name anyclaw \
  -p 8080:8080 \
  -v anyclaw-data:/data \
  --restart unless-stopped \
  <你的DockerHub用户名>/anyclaw-manager:latest
```

### 方式 B：指定公网访问地址

若需从外网访问，需设置 `ANYCLAW_API_URL`，供容器内的实例连接 API：

```bash
docker run -d \
  --name anyclaw \
  -p 8080:8080 \
  -v anyclaw-data:/data \
  -e ANYCLAW_API_URL=https://your-domain.com \
  --restart unless-stopped \
  <你的DockerHub用户名>/anyclaw-manager:latest
```

### 方式 C：配合 MySQL 与反向代理

```bash
# 若已通过 setup 配置过，可用环境变量跳过向导
docker run -d \
  --name anyclaw \
  -p 8080:8080 \
  -v anyclaw-data:/data \
  -e ANYCLAW_DB_DSN="user:pass@tcp(mysql-host:3306)/anyclaw?parseTime=true&charset=utf8mb4" \
  -e ANYCLAW_API_URL=https://your-domain.com \
  --restart unless-stopped \
  <你的DockerHub用户名>/anyclaw-manager:latest
```

### 方式 D：无本地持久化卷（Sealos/K8s 等）

所有配置已存数据库，只需通过环境变量提供启动参数即可，**无需挂载 /data**：

```bash
docker run -d \
  --name anyclaw \
  -p 8080:8080 \
  -e ANYCLAW_DB_DSN="user:pass@tcp(mysql-host:3306)/anyclaw?parseTime=true&charset=utf8mb4" \
  -e ANYCLAW_JWT_SECRET="随机32位以上字符串" \
  -e ANYCLAW_API_URL=https://your-domain.com \
  --restart unless-stopped \
  <你的DockerHub用户名>/anyclaw-manager:latest
```

- AI 渠道、SMTP、支付、金币等配置均在管理后台保存到 DB
- 首次部署需先访问 `/setup` 创建管理员，或通过 setup 向导完成

### 注册邮箱验证（可选）

启用后，用户注册需先获取邮箱验证码。管理员在「AI配置」页的「邮件服务」区配置 SMTP，保存后立即生效，可点击「测试连通性」验证。不配置时注册无需验证码。

---

## 五、首次使用

1. 访问 `http://你的服务器:8080`（或配置的域名）
2. 进入**初始化向导**，填写 MySQL 连接信息
3. 创建**管理员账号**
4. 按提示**重启容器**：`docker restart anyclaw`
5. 管理员登录后，在「服务器」中配置 SSH 主机（用于运行实例容器）

---

## 六、更新部署

```bash
# 1. 拉取最新镜像
docker pull <你的DockerHub用户名>/anyclaw-manager:latest

# 2. 停止并删除旧容器
docker stop anyclaw
docker rm anyclaw

# 3. 用新镜像启动（保持相同参数和卷）
docker run -d \
  --name anyclaw \
  -p 8080:8080 \
  -v anyclaw-data:/data \
  --restart unless-stopped \
  <你的DockerHub用户名>/anyclaw-manager:latest
```

或使用 `docker compose` 管理，便于一键更新。

---

## 七、迁移与换域名

换前端/域名后宠物会失联，因容器内记录的 API 地址未更新。可按以下步骤迁移：

1. **新环境**：用同一 MySQL 部署新服务，环境变量 `ANYCLAW_API_URL` 设为新域名
2. **管理后台**：登录后进入「AI配置」→「部署配置」，将 API 地址改为新域名并保存
3. **重新连接**：点击「重新连接实例」，系统会重启所有运行中宠物容器并注入新 API 地址
4. 完成后宠物即可恢复连接

API 地址也可在管理后台配置并保存到 DB，无需重启服务即可生效。

## 八、常见问题

| 问题 | 处理 |
|------|------|
| 镜像拉取 404 | 检查镜像名是否正确，私有仓库需 `docker login` |
| 首次访问无响应 | 确认 8080 端口已开放，防火墙/安全组放行 |
| 配置丢失 | 使用环境变量 `ANYCLAW_DB_DSN`、`ANYCLAW_JWT_SECRET` 时无需挂载卷；否则需 `-v anyclaw-data:/data` |
| 实例创建失败 | 管理员需在「服务器」中添加并启用至少一台 SSH 主机 |
| 换域名后宠物失联 | 见「七、迁移与换域名」 |

## 九、推荐生产架构

```
                    ┌─────────────────┐
                    │  Nginx / Caddy   │  ← 反向代理、HTTPS
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │ anyclaw-manager │  ← 8080
                    │  (API + 前端)   │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              │              │              │
       ┌──────▼──────┐ ┌─────▼─────┐ ┌─────▼─────┐
       │   MySQL     │ │  Host 1   │ │  Host 2   │  ← SSH 调度
       │             │ │ (Docker)  │ │ (Docker)  │
       └─────────────┘ └───────────┘ └───────────┘
```

- anyclaw-web 与 MySQL 可同机或分机部署
- 实例容器通过 SSH 调度到 Host 1、Host 2 等节点运行
- 调度器支持负载均衡（选实例数最少的主机）、宿主机容量上限、实例迁移、宿主机排空（维护前一键迁移该主机所有实例）
- AI 模型调度：同一模型配置多个渠道时自动负载均衡（选请求数最少的渠道），并定期保活探测各渠道

## 十、HTTPS 配置

**前提**：域名已解析到服务器 IP，80、443 端口已放行。

### 方案一：Caddy（推荐，自动申请证书）

```bash
# 安装
apt install -y debian-keyring debian-archive-keyring apt-transport-https curl
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | tee /etc/apt/sources.list.d/caddy-stable.list
apt update && apt install caddy

# 配置 /etc/caddy/Caddyfile（替换 your-domain.com）
cat > /etc/caddy/Caddyfile << 'EOF'
your-domain.com {
    reverse_proxy localhost:8080
}
EOF

systemctl reload caddy
```

### 方案二：Nginx + Certbot

```bash
apt install nginx certbot python3-certbot-nginx

# 配置 /etc/nginx/sites-available/anyclaw（替换 your-domain.com）
cat > /etc/nginx/sites-available/anyclaw << 'EOF'
server {
    listen 80;
    server_name your-domain.com;
    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
EOF

ln -s /etc/nginx/sites-available/anyclaw /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx

# 申请 HTTPS 证书
certbot --nginx -d your-domain.com
```

### 启动 anyclaw 时设置 API 地址

```bash
docker run -d \
  --name anyclaw \
  -p 8080:8080 \
  -v anyclaw-data:/data \
  -e ANYCLAW_API_URL=https://your-domain.com \
  --restart unless-stopped \
  <你的DockerHub用户名>/anyclaw-manager:latest
```

**注意**：`ANYCLAW_API_URL` 必须为 `https://`，否则 WebSocket 和容器连接会异常。
