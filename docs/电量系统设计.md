# AnyClaw 电量系统设计

## 一、概念

- **电量（能量）**：AI 使用的积分，等同于「能量」
- **宠物（AnyClaw）**：用户领养的实例，每个实例对应一只宠物
- **宠舍**：主界面，展示用户领养的所有宠物

## 二、规则

| 场景 | 消耗 | 说明 |
|------|------|------|
| 领养宠物 | 100 电量 | 创建新实例时扣除 |
| 每日维持 | 10 电量/天 | 每只宠物每天消耗 |
| 主人任务（AI 对话） | 按 token 消耗 | 每 1000 token 消耗 1 电量，不足 1000 按 1 计 |
| 任务门槛 | ≥5 电量 | 低于此值无法对话 |
| 永久消失 | 连续 3 天无电 | 宠物被删除 |

## 三、充值方式

1. **管理员充值**：后台为指定用户加电量
2. **邀请奖励**：邀请新用户注册，双方各得 50 电量

## 四、数据库

- `users.energy`：用户总电量
- `instances.energy`：宠物当前电量
- `instances.daily_consume`：每日消耗（默认 10）
- `instances.zero_energy_since`：首次变为 0 的时间
- `invitations`：邀请码与使用记录

## 五、API

- `POST /energy/invite`：获取邀请码
- `POST /energy/invite/use`：使用邀请码
- `POST /admin/energy/recharge`：管理员为用户充值（body: user_id, amount）
- `POST /admin/energy/daily`：执行每日消耗与清理（需定时调用，如 cron）
