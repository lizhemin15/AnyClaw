# AnyClaw 商业计划书

## 1. 项目概述

- **产品名**：AnyClaw  
- **定位**：基于 Open Claw 的轻量、可商用托管服务，让 Anyone 都能用上 Claw（精简版 Open Claw）。  
- **模式**：SaaS 订阅；用户付费后系统自动为其创建独立 Docker 容器实例，绑定账号，在网页端与 AnyClaw 交互。

---

## 2. 产品与功能

| 功能       | 说明 |
|------------|------|
| 注册/登录  | 邮箱或第三方登录，JWT 鉴权。 |
| 套餐与购买 | 按时长/实例数定价（如月付、年付、多实例包），支付成功后自动开通。 |
| 实例管理   | 查看我的实例、创建（随购买）、停止/启动、删除。 |
| 与 Claw 交互 | 网页控制台：对话、发指令、查看结果，请求经 API 转发到对应用户的 AnyClaw 容器。 |
| 运维侧     | 监控各宿主机与容器状态，调度新实例、迁移故障实例。 |

---

## 3. 资源与容量估算

### 3.1 单容器资源（依据你给的数据）

- 镜像体积：约 20MB+  
- 运行内存：约 10MB/容器  

### 3.2 单台 2GB 双核、220 元/年 服务器

| 项目         | 预留/估算 | 说明 |
|--------------|-----------|------|
| 系统 + 基础服务 | ~200MB   | 系统、Docker daemon、日志等。 |
| AnyClaw-API + DB | ~300MB | 轻量 API + PostgreSQL（或 SQLite）及连接。 |
| 调度/缓存等   | ~100MB   | 调度服务或 Agent、少量缓存。 |
| **可用于容器** | **~1.4GB** | 2GB − 约 600MB。 |
| **单机容器数** | **约 120～140 个** | 1.4GB ÷ 10MB/容器，取整并留余量；**建议单机上限设为 100～120**，便于预留突发与监控开销。 |

CPU（2 核）：Claw 轻量交互场景下，2 核可支撑百级容器；若后续有重计算，可再按 CPU 限流或降单机实例数。

磁盘：单镜像 20MB，100 个容器约 2GB 镜像层（共享层可复用），再加日志与数据，单机 10～20GB 磁盘通常足够。

**结论**：单台 2GB 双核机（220 元/年）可稳定运行 **约 100 个 AnyClaw 实例**；若压测显示 10MB 偏乐观，可按 15MB/容器重算，则约 **90 个/台**。

## 4. 调度与多机设计（灵活调度）

### 4.1 目标

- 新购实例自动选一台「有余量」的宿主机创建容器。  
- 单机故障时，可将该机上的实例标记为不可用并在其他机器重建（或等待恢复）。  
- 便于后续加机器、扩容。

### 4.2 设计要点

- **实例表**：每条记录对应一个用户实例，字段包含 `host`（宿主机标识）、`container_id`、`port`（若固定或动态分配）、`status`。  
- **宿主机表**：记录每台机器的地址、最大实例数、当前实例数、心跳时间；调度器按「当前数 < 最大数」且心跳正常选择主机。  
- **创建流程**：支付回调 → 写订单 → 调度器选 host → 在该 host 上 `docker run` 创建容器 → 写实例表（host, container_id, port）→ 用户可见「已开通」。  
- **单机容量**：每台 2GB 机器在配置里设 `max_instances = 100`（或 90）；调度时只选 `current < max_instances` 的机器。  
- **故障**：定时任务检查心跳；超时则将该 host 上所有实例标为「待恢复」或「待迁移」，并可选地在其他 host 上按实例列表逐个重建、更新实例表。  

这样可在多台 2GB 机器之间灵活分配实例，并按需加机、不绑定单点。

### 4.3 成本与规模（仅服务器）

| 场景           | 机器数 | 年成本（220 元/台） | 约实例数（100/台） |
|----------------|--------|----------------------|---------------------|
| MVP/试运行     | 1      | 220 元               | ~100                |
| 小规模         | 3      | 660 元               | ~300                |
| 扩展           | 10     | 2,200 元             | ~1,000              |

## 5. 定价与收入假设（示例）

仅作量级参考，需按市场与竞品再定：

- 月付：9.9～19.9 元/月/实例（或按「1 实例 + 若干对话量」打包）。  
- 年付：99～149 元/年/实例（约 8～9 折）。  

若单机 100 实例、均价 12 元/月/实例：

- 单机月收入约 1,200 元，年约 14,400 元；  
- 单机年服务器成本 220 元，毛利空间大；实际需扣支付通道费、域名、备份、人工等。

## 6. 风险与应对

| 风险           | 应对 |
|----------------|------|
| 单机宕机       | 调度层做心跳与实例状态标记，故障时在其他机器重建或提示用户「实例恢复中」。 |
| 支付争议/退款  | 明确退款策略；退款后调度层执行停用/删除实例并更新配额。 |
| 滥用（挖矿、爬虫等） | 容器限 CPU/内存、网络策略；必要时单实例限流或封禁。 |
| 合规           | 用户协议、隐私政策、数据留存与删除规则需符合目标市场法律。 |

---

## 7. 里程碑（建议）

1. **Phase 1**：单机 MVP — 官网（AnyClaw-Web）+ API（AnyClaw-API）+ 支付回调自动创建 1 个 AnyClaw 容器，网页与 Claw 交互打通。  
2. **Phase 2**：多实例与调度 — 支持多台 2GB 宿主机、调度器选机创建、实例列表与启停。  
3. **Phase 3**：稳定性与运营 — 心跳、故障迁移/恢复、监控告警、定价与套餐正式化。

---

## 8. 小结

- **容量**：单台 2GB 双核（220 元/年）可跑 **约 100 个** AnyClaw 容器（按 10MB/容器、预留约 600MB 给系统与 API）。  
- **调度**：用「宿主机表 + 实例表 + 调度器」在多机间分配与迁移实例，实现灵活、可扩展的调度。  
- **技术**：编译通过现有 GitHub Action 出 Docker 镜像；网站与后端按《技术架构设计》中的 AnyClaw-Web / AnyClaw-API 选型落地即可。
