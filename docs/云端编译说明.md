# 云端编译说明

所有构建均在 GitHub Actions 中完成，**本地无需安装 Go、Node、Docker 等环境**。

## 工作流

| 工作流 | 触发 | 产出 |
|--------|------|------|
| **Build All** | push / PR / 手动 | Server/API 二进制、Web 静态文件（Artifacts） |
| **Docker Build** | push / tag v* / 手动 | anyclaw-manager 镜像（Docker Hub） |

## 使用方式

### 1. 推送代码触发构建

```bash
git push origin main
```

在 GitHub Actions 页面查看构建状态，完成后可下载 Artifacts。

### 2. 手动触发

GitHub 仓库 → Actions → 选择工作流 → Run workflow

### 3. 下载构建产物

- **Build All**：Jobs 完成后，在对应 Job 的 Summary 中下载 Artifact
  - `anyclaw-server`：可执行文件
  - `anyclaw-api`：可执行文件
  - `anyclaw-web`：`dist/` 静态文件（可部署到 Nginx/OSS）

### 4. 使用 Docker 镜像

打 tag 或 push 后，Docker 工作流会构建并推送到 Docker Hub：

```
<你的DockerHub用户名>/anyclaw-manager:latest   # API + 前端合一，单端口访问
```

拉取并运行：

```bash
docker pull <你的DockerHub用户名>/anyclaw-manager:latest
docker run -p 8080:8080 -v anyclaw-data:/data <你的DockerHub用户名>/anyclaw-manager:latest
```

访问 `http://localhost:8080`，首次进入会引导配置 MySQL 和管理员账号。配置保存在 `/data/config.json`，需挂载卷持久化。

## 数据库（MySQL）

使用 MySQL 8.0+。首次 setup 时需已创建库：

```sql
CREATE DATABASE anyclaw CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

## 环境变量（可选）

- `ANYCLAW_CONFIG_PATH`：配置文件路径（默认 `/data/config.json`）
- `ANYCLAW_DB_DSN`：若已设置，跳过首次 setup，直接连接数据库
- `VITE_API_URL`：Web 构建时的 API 地址（默认 `http://localhost:8080`）

## 首次使用

1. 启动容器并挂载卷：`docker run -p 8080:8080 -v anyclaw-data:/data ...`
2. 访问 `http://localhost:8080`，自动进入初始化向导
3. 配置 MySQL 连接（需已创建 `anyclaw` 库）
4. 创建管理员账号
5. 按提示重启容器：`docker restart <容器名>`

管理员登录后，导航栏会显示「配置」「监控」「用户」「服务器」等入口。在「用户」中可添加用户、设置权限（普通用户/管理员）、充值金币；在「服务器」中配置 SSH 主机，实例创建时会在已启用的 Host 上通过 SSH 执行 `docker run`。
