# 容器环境与精简预制方案

## 现状分析

### 当前镜像层级

| 镜像 | 基础 | 预装 | 用途 | Skill 安装 | MCP (npx) | exec 跑 python |
|------|------|------|------|------------|-----------|----------------|
| **minimal** (Dockerfile / goreleaser) | alpine | ca-certificates, tzdata, curl | 宠物容器默认 | ✅ | ❌ | ❌ |
| **full** (Dockerfile.full) | node:24-alpine | python3, py3-pip, git, uv | 完整 MCP 支持 | ✅ | ✅ | ✅ |

### 各功能依赖

1. **Skill 安装**：纯 Go 实现
   - HTTP 从 Skillhub/ClawHub 下载 ZIP（默认 Skillhub 优先，国内加速）
   - `archive/zip` 解压（无需系统 unzip）
   - **minimal 镜像已支持**，无需额外依赖

2. **MCP 服务器**：需 Node.js
   - 常用配置：`npx -y @modelcontextprotocol/server-filesystem` 等
   - 依赖 `npx` → 需要 Node.js + npm

3. **exec 工具**：运行 shell 命令
   - 若需执行 `python` 脚本、`npm run` 等，需对应运行时
   - 当前 minimal 无 node/python

## 精简预制方案

### 方案 A：Dockerfile.medium（推荐）

在 minimal 与 full 之间增加「中等」镜像，只预装 Node.js，满足大部分 MCP 需求：

- 基础：`node:20-alpine`（比 node:24 更小、更稳定）
- 预装：ca-certificates, curl, git（skill 从 GitHub 安装时可选）
- **不预装**：python3, py3-pip, uv（减少体积）
- 体积：约 80–120MB（vs full 约 150–200MB）

适用：需要 MCP（filesystem、github 等），不需要 Python MCP 或 uvx 的场景。

### 方案 B：优化 Dockerfile.full

- 将 `node:24-alpine` 改为 `node:20-alpine`
- 若 MCP 仅用 npx，可去掉 uv/uvx
- 保留 python3 以支持 Python MCP 和 exec 中的 python 脚本

### 方案 C：双镜像策略（当前）

- `anyclaw/anyclaw:latest` → minimal（宠物默认）
- `anyclaw/anyclaw:full` → full（需 MCP 时在 Host 配置 docker_image）

管理员在「服务器」中为 Host 指定 `anyclaw/anyclaw:full` 即可使用完整功能。

## Skillhub 商店（预制）

- **配置**：默认 registry 已配置为 [Skillhub](https://skillhub.tencent.com) 优先（国内加速、合规），镜像为 clawhub.ai、clawhub.com
- **镜像**：Docker 镜像已预制 Skillhub，首次运行 `onboard` 生成的 config 即含 Skillhub；`/opt/anyclaw/config.example.json` 为参考副本
- 详见 [skillhub 安装说明](https://skillhub-1388575217.cos.ap-guangzhou.myqcloud.com/install/skillhub.md)

## 使用建议

| 场景 | 推荐镜像 |
|------|----------|
| 宠物容器、仅聊天 + Skill 安装 | minimal (默认) |
| 需要 MCP（filesystem、github 等） | medium 或 full |
| 需要 Python MCP / uvx / exec 跑 python | full |

## 预缓存 MCP 包（可选）

为加快首次 MCP 启动，可在构建时预拉取常用包：

```dockerfile
# 预缓存常用 MCP 包，首次启动无需下载
RUN npm cache add @modelcontextprotocol/server-filesystem@latest 2>/dev/null || true
```

或通过 volume 挂载 npm cache（docker-compose.full.yml 已配置 `anyclaw-npm-cache`）。
