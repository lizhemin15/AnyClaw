# AnyClaw 金币系统设计

## 一、概念

- **金币**：用户持有的积分，用于领养宠物和对话消耗
- **宠物（实例）**：用户领养的 AnyClaw 实例，每个实例对应一只宠物
- **宠舍**：主界面，展示用户领养的所有宠物

## 二、规则

| 场景 | 消耗 | 说明 |
|------|------|------|
| 领养宠物 | 100 金币 | 创建新实例时从用户扣除 |
| 主人任务（AI 对话） | 按 token 消耗 | 每 1000 token 约 1 金币，不足 1000 按 1 计 |
| 任务门槛 | ≥5 金币 | 低于此值无法发起对话 |

**说明**：已取消宠物级活力值、每日维持消耗、零活力删除等机制，全部改为用户级金币。

## 三、充值方式

1. **管理员充值**：管理后台「用户管理」为指定用户充值金币
2. **邀请奖励**：邀请新用户注册，双方各得金币（数量可配置）
3. **激活码兑换**：用户输入激活码兑换金币

## 四、数据库

- `users.energy`：用户金币余额
- `usage_log`：对话消耗记录（instance_id, user_id, tokens, coins_cost）
- `invitations`：邀请码与使用记录
- `activation_codes`：激活码（生成、核销）

## 五、API

- `POST /energy/invite`：获取邀请码
- `POST /energy/invite/use`：使用邀请码
- `POST /energy/redeem-code`：用户兑换激活码
- `GET /me/usage`：用户消耗记录
- `GET /admin/energy/users`：管理员列出用户
- `POST /admin/energy/users`：管理员添加用户
- `PUT /admin/energy/users/:id`：管理员修改用户（权限、金币）
- `POST /admin/energy/users/:id/recharge`：管理员为用户充值
- `POST /admin/activation-codes`：生成激活码
- `POST /admin/activation-codes/verify`：核销前校验
- `POST /admin/activation-codes/redeem`：代用户兑换
